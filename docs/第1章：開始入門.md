# 第一章：開始入門

## 建立新專案

我們建議使用 `pnpm` 作為您的套件管理器，因為它比 `npm` 或 `yarn` 更快速且效率更高。如果您尚未安裝 `pnpm`，可以透過執行以下指令全域安裝：

```bash
npm install -g pnpm
```

要建立一個 Next.js 應用程式，請開啟您的終端機，切換到您希望存放專案的資料夾，然後執行以下指令：

```bash
npx create-next-app@latest nextjs-dashboard --example "https://github.com/vercel/next-learn/tree/main/dashboard/starter-example" --use-pnpm
```

此指令使用了 `create-next-app`，這是一個 Command Line Interface (CLI) 工具，用於為您設定 Next.js 應用程式。在上述指令中，您也使用了 `--example` 旗標來指定本課程的入門範例。

## 探索專案結構

與從零開始編寫程式碼的教學不同，本課程的大部分程式碼已預先為您寫好。這更能反映真實世界的開發情境，因為您很可能會在現有的程式碼庫上工作。

我們的目標是幫助您專注於學習 Next.js 的主要功能，而無需編寫所有應用程式碼。

安裝完成後，請在您的程式碼編輯器中開啟專案，並進入 `nextjs-dashboard` 資料夾：

```bash
cd nextjs-dashboard
```

讓我們花點時間探索這個專案。

### 資料夾結構

您會注意到專案具有以下資料夾結構：

本專案的資料夾結構主要包含 `app`、`public` 以及一些設定檔。

- **`/app`**: 包含您應用程式的所有路由、元件和邏輯。這將是您主要工作的地方。
  - **`/app/lib`**: 包含您應用程式中使用的函數，例如可重複使用的工具函數和資料獲取函數。
  - **`/app/ui`**: 包含您應用程式的所有 UI 元件，例如卡片、表格和表單。為節省您的時間，我們已為這些元件預先設定了樣式。
- **`/public`**: 包含您應用程式的所有靜態資產，例如圖片。
- **設定檔 (Config Files)**: 您也會在應用程式的根目錄中看到 `next.config.ts` 等設定檔。這些檔案大多數在您使用 `create-next-app` 建立新專案時自動產生並預先設定。在本課程中，您無需修改它們。

請隨意探索這些資料夾，即使您暫時不完全理解所有程式碼的功能也無需擔心。

### 預留資料 (Placeholder Data)

當您建構使用者介面時，有一些預留資料會很有幫助。如果資料庫或 API 尚未準備就緒，您可以：

- 使用 JSON 格式或 JavaScript 物件的預留資料。
- 使用第三方服務，例如 mockAPI。

對於此專案，我們在 `app/lib/placeholder-data.ts` 中提供了一些預留資料。檔案中的每個 JavaScript 物件都代表資料庫中的一個表格。例如，以下是 `invoices` 表格的資料範例：

`/app/lib/placeholder-data.ts`

```typescript
const invoices = [
  {
    customer_id: customers[0].id,
    amount: 15795,
    status: "pending",
    date: "2022-12-06",
  },
  {
    customer_id: customers[1].id,
    amount: 20348,
    status: "pending",
    date: "2022-11-14",
  },
  // ...
];
```

在設定資料庫的章節中，您將會使用這些資料來填充您的資料庫（即用一些初始資料來填充它）。

### TypeScript

您可能還會注意到大多數檔案都帶有 `.ts` 或 `.tsx` 的副檔名。這是因為此專案是使用 TypeScript 編寫的。我們希望建立一個能反映現代網路趨勢的課程。

如果您不了解 TypeScript 也沒關係——我們會在需要時提供 TypeScript 程式碼片段。

現在，請查看 `/app/lib/definitions.ts` 檔案。在這裡，我們手動定義了將從資料庫返回的型別。例如，`invoices` 表格具有以下型別定義：

`/app/lib/definitions.ts`

```typescript
export type Invoice = {
  id: string;
  customer_id: string;
  amount: number;
  date: string;
  // 在 TypeScript 中，這稱為字串聯合型別（string union type）。
  // 這表示 "status" 屬性只能是 'pending' 或 'paid' 這兩個字串之一。
  status: "pending" | "paid";
};
```

透過使用 TypeScript，您可以確保不會意外地將錯誤的資料格式傳遞給您的元件或資料庫，例如將字串傳遞給發票金額而不是數字。

> **如果您是 TypeScript 開發者：**
> 我們是手動聲明資料型別的，但為了更好的型別安全，我們推薦使用 Prisma 或 Drizzle，它們可以根據您的資料庫 Schema 自動生成型別。
> Next.js 會偵測您的專案是否使用 TypeScript，並自動安裝必要的套件和配置。Next.js 還為您的程式碼編輯器提供了 TypeScript 插件，以幫助實現自動完成和型別安全。

## 執行開發伺服器

執行 `pnpm i` 來安裝專案的套件：

```bash
pnpm i
```

接著執行 `pnpm dev` 來啟動開發伺服器：

```bash
pnpm dev
```

`pnpm dev` 會在 port 3000 上啟動您的 Next.js 開發伺服器。讓我們檢查它是否正常運作。

在您的瀏覽器中開啟 `http://localhost:3000`。您的首頁應該會如下圖所示，此時它是刻意未經樣式處理的：
